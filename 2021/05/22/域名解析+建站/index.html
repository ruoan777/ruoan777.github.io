域名解析+建站

 2021-05-22 10:55:06  

域名解析基本原理如下图

正式步骤

  1. 自行申请域名,申请vps(下面以centos为例)

  2. 修改系统时间为北京时间

    date -R
timedatectl set-local-rtc 1
timedatectl set-timezone Asia/Shanghai

  3. 安装相关依赖

    yum makecache
yum install curl
yum install netcat

  4. 安装dat和release.sh

    curl -O https://raw.githubusercontent.com/v2fly/fhs-install-v2ray/master/install-release.sh
curl -O https://raw.githubusercontent.com/v2fly/fhs-install-v2ray/master/install-dat-release.sh

bash install-release.sh
bash install-dat-release.sh

  5. 编辑配置文件

    cd /usr/local/etc/v2ray
vim config.json

{
  "inbounds": [
    {
      "port": 443,
      "protocol": "vmess",    
      "settings": {
        "clients": [
          {
            "id": "这里是随机生成的uuid",
            "alterId": 16
          }
        ]
      },
      "streamSettings": {
        "network": "ws",
        "security": "tls",
        "tlsSettings": {
          "certificates": [
            {
              "certificateFile": "/etc/v2ray/v2ray.crt",
              "keyFile": "/etc/v2ray/v2ray.key"
            }
          ]
        }
      }
    }
  ],
  "outbounds": [
    {
      "protocol": "freedom",
      "settings": {}
    }
  ]
}

  6. 安装acem.sh证书生成工具

    case 1: centos7 
yum install socat
yum install netcat

case 2: debian
apt-get install openssl cron socat curl
apt-get -y install netcat

curl  https://get.acme.sh | sh -s email=xxx@gmail.com
source ~/.bashrc

  7. 生成v2ray目录

    cd /etc/  
mkdir v2ray   
cd ~

  8. 临时关闭防火墙

    systemctl stop firewalld.service
systemctl disable firewalld.service

  9. 生成证书

    acme.sh --register-account -m xxx@gmail.com
sudo ~/.acme.sh/acme.sh --issue -d 这里是域名 --standalone -k ec-256

    这一步如果有如下报错，说明防火墙可能没有关闭, 如果使用的centos系统，关闭防火墙就可以成功获取证书，即还是要执行上一步

  10. 证书和密钥安装到指定路径

    cd etc/v2ray  
sudo ~/.acme.sh/acme.sh --installcert -d 这里是域名 --fullchainpath /etc/v2ray/v2ray.crt --keypath /etc/v2ray/v2ray.key --ecc

记得要修改文件权限
chmod 644 /etc/v2ray/v2ray.key

  11. 查询tls开启状态
    查询tls

  12. 重启且设置且开机启动

    sudo systemctl restart v2ray
systemctl enable v2ray

查看当前状态
sudo systemctl status -l v2ray

  13. 防火墙设置

sudo ufw status
sudo ufw allow 443
sudo ufw enable
sudo ufw reload
Centos7默认安装了firewalld，如果没有安装的话，可以使用  
yum install firewalld firewalld-config  
进行安装。

开启防火墙
systemctl start firewalld 
设置开机启动
systemctl enable firewalld
查看当前防火墙状态
systemctl status firewalld

在指定区域打开端口（记得重启防火墙）
firewall-cmd --zone=public --add-port=80/tcp  --permanent
firewall-cmd --zone=public --add-port=443/tcp  --permanent
firewall-cmd --zone=public --list-ports
重启防火墙
firewall-cmd --reload

  1. 卸载

    bash install-release.sh --remove

  2. 一些踩坑

  • uuid不能乱生成，不然会报无法解析成功的错
  • 服务端端口有可能不通，会报无法连接，这时候去服务端开启对应端口就行了

  1. ssh 免密登陆
    如果本地机器已经有了公私钥，那么直接执行下面的命令即可，XXX为目标远程服务器的ip，回车后会要求输入远程服务器的密码。
    ssh-copy-id -i ~/.ssh/id_rsa.pub root@XXX

如果本地机器还没有公私钥，则需要先执行下面的命令生成密钥。
ssh-keygen

附ssh 快捷登陆的配置，注意配置的目录是在： ~/.ssh 的config文件。那么之后就可以使用 ssh ruoan 直接登陆远程服务器了。
➜ .ssh cat config
Host ruoan
HostName XXX
User root
Port 22
Host github.com
User git
ProxyCommand nc -v -x 127.0.0.1:1080 %h %p
